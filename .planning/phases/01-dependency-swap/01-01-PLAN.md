  <name>Task 2: Update tsconfig.json — add moduleResolution node16</name>
  <files>tsconfig.json</files>
  <read_first>
    - tsconfig.json — read current state to find the exact commented moduleResolution line to replace
  </read_first>
  <action>
In tsconfig.json, find the commented-out moduleResolution line:
```
    // "moduleResolution": "node10",                     /* Specify how TypeScript looks up a file from a given module specifier. */
```

Replace it with an active (uncommented) setting:
```
    "moduleResolution": "node16",                        /* Required for Vercel AI SDK — reads package.json exports field. Compatible with module: commonjs. */
```

**Why node16 and NOT bundler:** TypeScript 5+/6 enforces that `moduleResolution: "bundler"` requires a modern ES module format (`esnext`, `preserve`, etc.). This project uses `"module": "commonjs"` which is incompatible with `"bundler"`. `"node16"` is the correct and only valid choice for CJS projects that need to resolve package `exports` fields.

Do NOT change any other tsconfig.json setting.
  </action>
  <verify>

node -e "JSON.parse(require('fs').readFileSync('tsconfig.json','utf8').replace(/\/\/[^\n]*/g,'').replace(/\/\*[\s\S]*?\*\//g,'')); console.log('OK: tsconfig is valid')" 2>&1
    </automated>
  </verify>
  <acceptance_criteria>

  </acceptance_criteria>
